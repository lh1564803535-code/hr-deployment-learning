## 💡 Commit 信息模板

为了让你的 GitHub 提交记录更清晰，建议使用以下模式：

### 基础格式
```
[Emoji] [主题]: [简短描述]
```

### 推荐 Emoji
- 📚 学习笔记/文档更新
- 🏗️ 架构/结构更新
- 🐳 Docker 相关
- 📝 Day N 的学习记录
- 🔧 问题修复
- ⚡ 性能优化
- 🚀 新功能/新知识

### 具体示例

```bash
# Day 1 - Docker 基础
git commit -m "📚 Day 1: Docker基础概念和镜像/容器区别"

# Day 2 - Dockerfile
git commit -m "🏗️ Day 2: Dockerfile编写和多阶段构建学习"

# Day 3 - Docker Compose
git commit -m "🐳 Day 3: Docker Compose多容器编排实践"

# Day 4 - 部署
git commit -m "🚀 Day 4: 完整三层应用容器化部署"

# 问题排查
git commit -m "🔧 Day 5: 常见Docker问题排查笔记"

# 学习总结
git commit -m "📝 学习总结: Docker完整学习30天路线"
```

## 📞 常见问题

### Q: 如何更新学习笔记？
**A:** 编辑 `learning-notes/daily-notes.md` 文件，添加新的一天。

### Q: 忘记了 Commit 信息？
**A:** 没关系，随意写就可以，只要 push 到 GitHub 就有记录。

### Q: 想要修改已提交的内容？
**A:** 
```bash
git log --oneline  # 查看提交历史
git revert HEAD    # 撤销最后一次提交（推荐）
```

### Q: Emoji 显示乱码？
**A:** 这是编码问题，不影响功能，可以在 GitHub 网页上正常显示。
